= Lab 1: Workshop Environment Overview
:navtitle: Environment Overview

== Overview

This workshop provides a pre-configured OpenShift environment with foundational infrastructure services.
You will deploy the application platform services (DevSpaces, Vault, Tekton, Argo CD) and build a complete CI/CD pipeline.

== Pre-Deployed Infrastructure

The following services are already installed and configured:

=== Red Hat Build of Keycloak (RHBK)

Centralized SSO/OIDC provider for GitLab, Quay, and other services.

* **URL:** {keycloak_url}
* **Realm:** `etx-lab`
* **Available Users:**
  - `platform-admin` / `openshift` (administrator access)
  - `platform-user` / `openshift` (standard user)
  - `platform-developer` / `openshift` (developer user)

TIP: Use these credentials to log into GitLab and Quay via the OIDC/OAuth buttons.

=== GitLab CE

Source control and Git repository hosting.

* **URL:** {gitlab_url}
* **Authentication:** Keycloak OIDC (click "Sign in with OpenID Connect")
* **Seeded Repositories:**
  - `software-factory/etx_app_base_app` - Your application workspace (read/write)

=== Quay Container Registry

Enterprise container registry for storing and distributing images.

* **URL:** {quay_url}
* **Authentication:** Keycloak OIDC or local account
* **Purpose:** External image registry for production-grade workflows

NOTE: For this lab, you'll primarily use the OpenShift internal registry. Quay is available for multi-cluster promotion scenarios.

=== OpenShift GitOps (Argo CD)

A pre-deployed GitOps operator manages infrastructure automation.

* **Instance:** `openshift-gitops` namespace
* **Purpose:** Infrastructure automation (GitLab, Keycloak, Quay deployments)
* **GitOps Repository:** `etx-gitops` - Contains infrastructure manifests managed by the bootstrap process

IMPORTANT: You will deploy your **own** Argo CD instance in Lab 1 Step 5 for application deployments. The `openshift-gitops` instance is reserved for infrastructure only.

== Accessing GitLab with Keycloak

. Open {gitlab_url}
. Click btn:[Sign in with OpenID Connect]
. You will be redirected to Keycloak
. Log in with one of the provided users (e.g., `platform-developer` / `openshift`)
. You will be redirected back to GitLab and logged in

The first time you log in, GitLab will create a user profile linked to your Keycloak identity.

== Clone Your Repository

Once logged into GitLab, clone the ETX App Base application:

[source,bash]
----
git clone {gitlab_url}/software-factory/etx_app_base_app.git
cd etx_app_base_app
----

NOTE: The repository name is `etx_app_base_app` and Kubernetes resources will be named `etx-app-base-app-*` for consistency.

== Verify Cluster Access

Confirm you're logged into the primary cluster:

[source,bash]
----
oc whoami --show-server
# Should show: https://api.cluster-{guid}.dynamic.redhatworkshops.io:6443

oc project etx-app-dev
# Should switch to: etx-app-dev namespace
----
