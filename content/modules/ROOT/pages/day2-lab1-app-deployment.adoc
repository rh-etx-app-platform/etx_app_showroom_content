= Lab 1: Application Deployment with Argo CD
:navtitle: Application Deployment

== Overview

Deploy your own Argo CD instance to manage application deployments using a GitOps workflow.
The CI pipeline from the previous section pushes container images; Argo CD watches a GitOps repository and syncs Kubernetes manifests to the cluster.

IMPORTANT: You will deploy your **own** Argo CD instance in the `parasol-dev` namespace. The pre-deployed OpenShift GitOps operator in `openshift-gitops` is used for infrastructure automation only and is separate from your application deployment workflows.

== Install OpenShift GitOps Operator

The OpenShift GitOps operator is already installed cluster-wide from the infrastructure setup.
Verify it's available:

[source,bash]
----
oc get csv -n openshift-operators | grep gitops
----

You should see `openshift-gitops-operator` with phase `Succeeded`.

== Deploy Your Argo CD Instance

Create an Argo CD instance in your namespace for managing application deployments:

[source,yaml]
----
apiVersion: argoproj.io/v1beta1
kind: ArgoCD
metadata:
  name: argocd
  namespace: parasol-dev
spec:
  server:
    autoscale:
      enabled: false
    grpc:
      ingress:
        enabled: false
    ingress:
      enabled: false
    resources:
      limits:
        cpu: 500m
        memory: 256Mi
      requests:
        cpu: 125m
        memory: 128Mi
    route:
      enabled: true
      tls:
        termination: edge
        insecureEdgeTerminationPolicy: Redirect
    service:
      type: ClusterIP
  grafana:
    enabled: false
    ingress:
      enabled: false
    route:
      enabled: false
  prometheus:
    enabled: false
    ingress:
      enabled: false
    route:
      enabled: false
  initialSSHKnownHosts: {}
  sso:
    provider: dex
    dex:
      openShiftOAuth: true
      resources:
        limits:
          cpu: 500m
          memory: 256Mi
        requests:
          cpu: 250m
          memory: 128Mi
  rbac:
    defaultPolicy: 'role:readonly'
    policy: |
      g, system:cluster-admins, role:admin
      g, cluster-admins, role:admin
    scopes: '[groups]'
  repo:
    resources:
      limits:
        cpu: 1000m
        memory: 1Gi
      requests:
        cpu: 250m
        memory: 256Mi
  resourceExclusions: |
    - apiGroups:
      - tekton.dev
      clusters:
      - '*'
      kinds:
      - TaskRun
      - PipelineRun
  controller:
    processors: {}
    resources:
      limits:
        cpu: 2000m
        memory: 2Gi
      requests:
        cpu: 250m
        memory: 1Gi
  ha:
    enabled: false
    resources:
      limits:
        cpu: 500m
        memory: 256Mi
      requests:
        cpu: 250m
        memory: 128Mi
----

Apply the ArgoCD CR:

[source,bash]
----
oc apply -f argocd.yaml
----

Wait for Argo CD to be ready (this may take 3-5 minutes):

[source,bash]
----
oc get pods -n parasol-dev -w
----

Press Ctrl+C when you see all `argocd-*` pods showing `1/1 Ready`.

=== Dev Environment Manifests

Create `environments/dev/namespace.yaml`:

[source,yaml]
----
apiVersion: v1
kind: Namespace
metadata:
  name: parasol-dev
----

Create `environments/dev/deployment.yaml`:

[source,yaml]
----
apiVersion: apps/v1
kind: Deployment
metadata:
  name: parasol-insurance
  namespace: parasol-dev
  labels:
    app: parasol-insurance
  annotations:
    instrumentation.opentelemetry.io/inject-java: "true" # <1>
spec:
  replicas: 1
  selector:
    matchLabels:
      app: parasol-insurance
  template:
    metadata:
      labels:
        app: parasol-insurance
      annotations:
        instrumentation.opentelemetry.io/inject-java: "true" # <1>
    spec:
      containers:
        - name: parasol-insurance
          image: image-registry.openshift-image-registry.svc:5000/parasol-dev/parasol-insurance:latest # <2>
          ports:
            - containerPort: 8080
          resources:
            requests:
              cpu: 250m
              memory: 512Mi
            limits:
              cpu: "1"
              memory: 1Gi
----
<1> Enables OpenTelemetry auto-instrumentation. The Instrumentation CR created in the xref:day2-lab1-supporting-services.adoc[Supporting Services] section will inject the Java agent automatically.
<2> The CI pipeline updates this image tag after a successful build.

Create `environments/dev/service.yaml`:

[source,yaml]
----
apiVersion: v1
kind: Service
metadata:
  name: parasol-insurance
  namespace: parasol-dev
spec:
  selector:
    app: parasol-insurance
  ports:
    - port: 8080
      targetPort: 8080
----

Create `environments/dev/route.yaml`:

[source,yaml]
----
apiVersion: route.openshift.io/v1
kind: Route
metadata:
  name: parasol-insurance
  namespace: parasol-dev
spec:
  to:
    kind: Service
    name: parasol-insurance
  port:
    targetPort: 8080
  tls:
    termination: edge
----

Commit and push the dev environment:

[source,bash]
----
git add environments/dev/
git commit -m "Add dev environment manifests"
git push origin main
----

== Create the Argo CD Application

Grant Argo CD permission to manage resources in the `parasol-dev` namespace:

[source,bash]
----
oc adm policy add-role-to-user admin \
  system:serviceaccount:parasol-dev:argocd-application-controller \
  -n parasol-dev
----

Create an Argo CD Application that watches the `dev` environment:

[source,yaml]
----
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: parasol-dev
  namespace: parasol-dev
spec:
  project: default
  source:
    repoURL: http://gitlab.apps.cluster-cgd7r.dynamic.redhatworkshops.io/student/parasol-gitops.git
    targetRevision: main
    path: environments/dev
  destination:
    server: https://kubernetes.default.svc
    namespace: parasol-dev
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=false # <1>
----
<1> Namespace already exists, so don't create it

Apply it:

[source,bash]
----
oc apply -f parasol-dev-app.yaml
----

== Integrate CI and GitOps

The CI pipeline needs to update the image tag in the GitOps repository after a successful build.
Add a final task to the pipeline that commits the new image tag:

[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Task
metadata:
  name: update-gitops-repo
  namespace: parasol-dev
spec:
  params:
    - name: gitops-repo-url
      type: string
    - name: image-tag
      type: string
    - name: environment
      type: string
      default: dev
  workspaces:
    - name: source
  steps:
    - name: update-image
      image: quay.io/devfile/universal-developer-image:ubi9-latest
      script: |
        #!/bin/bash
        set -e
        git clone $(params.gitops-repo-url) /tmp/gitops
        cd /tmp/gitops/environments/$(params.environment)

        # Update the image tag in the deployment manifest
        sed -i "s|image:.*parasol-insurance:.*|image: image-registry.openshift-image-registry.svc:5000/parasol-dev/parasol-insurance:$(params.image-tag)|" deployment.yaml

        git config user.email "pipeline@openshift.local"
        git config user.name "Tekton Pipeline"
        git add .
        git commit -m "Update parasol-insurance image to $(params.image-tag)"
        git push
----

Apply the task:

[source,bash]
----
oc apply -f update-gitops-task.yaml
----

Update the pipeline to include this task at the end:

[source,yaml]
----
    - name: update-gitops
      runAfter: [build-image]
      taskRef:
        name: update-gitops-repo
      params:
        - name: gitops-repo-url
          value: http://gitlab.apps.cluster-cgd7r.dynamic.redhatworkshops.io/student/parasol-gitops.git
        - name: image-tag
          value: $(params.git-revision)
        - name: environment
          value: dev
      workspaces:
        - name: source
          workspace: shared-workspace
----

When the pipeline pushes the updated manifest, Argo CD detects the change and automatically syncs the new image to the cluster.

=== Add Secondary Cluster to Argo CD

Get the secondary cluster API endpoint and token:

[source,bash]
----
# Get secondary cluster API URL
SECONDARY_API="https://api.cluster-d4f7p.dynamic.redhatworkshops.io:6443"

# Login to secondary cluster
oc login $SECONDARY_API -u admin -p G6VpeK3U6n8LpyhC

# Create service account for Argo CD
oc create serviceaccount argocd-manager -n kube-system

# Grant cluster-admin (for lab purposes only)
oc create clusterrolebinding argocd-manager-binding \
  --clusterrole=cluster-admin \
  --serviceaccount=kube-system:argocd-manager

# Get the service account token
SA_TOKEN=$(oc create token argocd-manager -n kube-system --duration=8760h)

# Switch back to primary cluster
oc login https://api.cluster-cgd7r.dynamic.redhatworkshops.io:6443 -u admin -p r0rDrvTWeTAbVk7L
----

Add the secondary cluster to Argo CD:

[source,bash]
----
# Create secret with cluster credentials
cat <<EOF | oc apply -f -
apiVersion: v1
kind: Secret
metadata:
  name: cluster-d4f7p
  namespace: parasol-dev
  labels:
    argocd.argoproj.io/secret-type: cluster
type: Opaque
stringData:
  name: cluster-d4f7p
  server: https://api.cluster-d4f7p.dynamic.redhatworkshops.io:6443
  config: |
    {
      "bearerToken": "${SA_TOKEN}",
      "tlsClientConfig": {
        "insecure": true
      }
    }
EOF
----

Verify the cluster was added:

[source,bash]
----

# You should see cluster-d4f7p listed
----

== Promotion Workflow

To promote an image from dev to stage:

. CI pipeline builds and pushes image to internal registry with commit SHA tag
. CI pipeline tags the same image in Quay with a `stable` tag:
+
[source,bash]
----
# Example: retag and push to Quay
skopeo copy \
  docker://image-registry.openshift-image-registry.svc:5000/parasol-dev/parasol-insurance:abc123 \
  docker://quay-registry-quay.apps.cluster-cgd7r.dynamic.redhatworkshops.io/parasol/parasol-insurance:stable
----
+
. Update `environments/stage/deployment.yaml` image tag to `stable`
. Push to GitOps repository
. Argo CD automatically syncs to secondary cluster
