= Lab 2: AI-assisted development with Red Hat Developer Lightspeed
:navtitle: Developer Lightspeed Configuration

== Overview

Red Hat Developer Lightspeed brings generative AI capabilities directly into your development environment, helping you understand migration patterns, generate code snippets, and receive context-aware recommendations as you modernize the Parasol application.

In this module, you'll configure the Migration Toolkit for Applications extension in OpenShift Dev Spaces and integrate it with Red Hat's Model-as-a-Service platform to enable AI-assisted application modernization.

== Learning objectives

By the end of this module, you'll be able to:

* Install and configure the Migration Toolkit for Applications extension in OpenShift Dev Spaces
* Register an application with Red Hat Model-as-a-Service (MaaS) platform
* Generate and securely configure API keys for Red Hat Developer Lightspeed
* Validate AI-assisted development capabilities within your workspace

== Exercise 1: Install MTA extension in Dev Spaces

Parasol Insurance needs to accelerate the application modernization process.
You'll start by installing the MTA extension in your Dev Spaces workspace to bring migration analysis capabilities directly into the IDE.

=== Access your Dev Spaces workspace

. Open the OpenShift Dev Spaces console:
+
[source,role="execute"]
----
echo "Dev Spaces URL: {devspaces_url}"
----

. Log in with your OpenShift credentials if prompted

. Open your existing `parasol-insurance-lab` workspace
+
If you don't have a workspace yet, create one from the GitLab repository:
+
[source,role="execute"]
----
# Workspace will be created from: {gitlab_url}/student/parasol-insurance-lab
----

=== Install the MTA extension

. In your Dev Spaces workspace, click the **Extensions** icon in the left sidebar (or press `Ctrl+Shift+X`)

. In the search box, type:
+
----
Migration Toolkit for Applications
----

. Locate the **Migration Toolkit for Applications (MTA)** extension published by Red Hat

. Click btn:[Install]

. Wait for the installation to complete
+
The extension icon will appear in the Activity Bar on the left side of the workspace.

=== Verify the installation

. Click the **MTA** icon in the Activity Bar

. You should see the MTA Analysis view with options to:
+
* Create new analysis configurations
* Configure Generative AI settings
* View analysis results

. Verify the extension is active:
+
The MTA view displays "No configurations found" -- this is expected for a fresh installation.

image::mta-extension-installed.png[MTA extension installed in Dev Spaces,link=self,window=blank,width=700]

[TIP]
====
If the extension doesn't appear, refresh the workspace by pressing `F1` and selecting **Developer: Reload Window**.
====

== Exercise 2: Configure Red Hat Developer Lightspeed

With the MTA extension installed, you need to connect it to Red Hat's AI services to enable Developer Lightspeed capabilities.
This requires registering an application in the Red Hat Model-as-a-Service platform and obtaining an API key.

=== Register with Red Hat MaaS platform

. Open the Red Hat Model-as-a-Service portal in a new browser tab:
+
link:https://maas.apps.prod.rhoai.rh-aiservices-bu.com/[Red Hat MaaS Portal^]

. Sign in with your Red Hat credentials
+
Use your Red Hat account associated with your OpenShift environment.

. Navigate to **Apps and API Keys** from the dashboard

. Click btn:[Create New Application]

. Fill in the application details:
+
* **Application Name**: `Parasol-Lightspeed-Dev`
* **Description**: `AI-assisted modernization for Parasol Insurance`

. Select the AI model service:
+
* **Model**: `Llama-4-Scout`
+
This model is optimized for code analysis and migration assistance.

. Click btn:[Create Application]

=== Generate API key

. After creating the application, you'll see the application details page

. Click btn:[Generate API Key]

. Copy the generated API key immediately
+
[IMPORTANT]
====
The API key is displayed only once.
Store it securely -- you'll need it in the next step.
====

. Save the API key to a secure location (password manager or encrypted file)

=== Configure GenAI settings in MTA

. Return to your Dev Spaces workspace

. Click the **MTA** icon in the Activity Bar

. In the MTA Analysis view, click **Configure GenAI** (gear icon)

. Create a new file named `genai-config.yaml` in your workspace root with the following content:
+
[source,yaml]
----
environment:
  ALLOW_INSECURE: "true"
models:
  RedHat-MaaS: &active
    environment:
      OPENAI_API_KEY: "<your-api-key-here>"
    provider: ChatOpenAI
    args:
      model: llama-4-scout-17b-16e-w4a16
      temperature: 0.2
      configuration:
        baseURL: "https://llama-4-scout-17b-16e-w4a16-maas-apicast-production.apps.prod.rhoai.rh-aiservices-bu.com:443/v1"
active: *active
----

. Replace `<your-api-key-here>` with the API key you generated in the previous step

. Save the file

. In the MTA view, click **Load GenAI Configuration**

. Select the `genai-config.yaml` file you just created

. Wait for the configuration to load
+
You should see a confirmation message: "GenAI configuration loaded successfully"

=== Verify GenAI configuration

. Check the MTA view for the GenAI status indicator

. The status should show:
+
----
GenAI: Configured ✓
Model: Llama-4-Scout
Status: Connected
----

. Test the AI assistance by right-clicking on a Java file in your workspace and selecting **MTA: Analyze with AI**
+
If the integration is working, you'll see AI-powered suggestions appear in the Problems panel.

image::genai-configured.png[GenAI successfully configured,link=self,window=blank,width=700]

[NOTE]
====
The GenAI configuration file contains sensitive credentials.
Add `genai-config.yaml` to your `.gitignore` file to prevent accidentally committing it to version control.
====
