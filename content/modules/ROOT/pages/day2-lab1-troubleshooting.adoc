= Lab 1: Troubleshooting Guide
:navtitle: Troubleshooting

*Quick reference for common issues during Lab 1.*

== Pipeline Issues

=== Git Clone Failures

**Symptom:** Pipeline fails at `clone` task with authentication errors.

**Solutions:**

[source,bash]
----
# For GitHub: Create Personal Access Token
# Settings → Developer settings → Personal access tokens → Generate new token
# Scopes: repo

# For GitLab: Use workshop credentials
# Username: {user}
# Password: {password}
----

=== Build Failures

**Symptom:** Maven build fails or buildah errors.

**Check:**

[source,bash]
----
# Test Maven build locally
cd ~/parasol-insurance-lab
./mvnw clean package -DskipTests

# Check buildah logs
oc logs <build-pod-name> -n parasol-dev
----

=== Image Push Failures

**Symptom:** Cannot push to registry.

**Solution:**

[source,bash]
----
# Verify ServiceAccount has permissions
oc adm policy add-role-to-user edit \
  system:serviceaccount:parasol-dev:pipeline -n parasol-dev

oc adm policy add-role-to-user system:image-builder \
  system:serviceaccount:parasol-dev:pipeline -n parasol-dev
----

== Deployment Issues

=== Pods Not Starting

**Symptom:** Pods in `CrashLoopBackOff` or `ImagePullBackOff`.

**Diagnosis:**

[source,bash]
----
# Check pod status
oc get pods -n parasol-dev

# View pod logs
oc logs <pod-name> -n parasol-dev --tail=50

# Describe pod for events
oc describe pod <pod-name> -n parasol-dev
----

**Common causes:**

* **ImagePullBackOff:** Image not built yet - run CI pipeline
* **CrashLoopBackOff:** Check logs for missing dependencies

=== Kafka Connection Issues

**Symptom:** Application fails with Kafka errors.

**Check:**

[source,bash]
----
# Verify Kafka cluster is ready
oc get kafka parasol-kafka -n parasol-dev

# Check Kafka pods
oc get pods -n parasol-dev | grep kafka

# Verify Kafka topic
oc get kafkatopic intake -n parasol-dev
----

**Fix:** Ensure AMQ Streams operator is installed and Kafka cluster is `Ready`.

=== Database Connection Issues

**Symptom:** Application cannot connect to PostgreSQL.

**Check:**

[source,bash]
----
# Check database pod
oc get pods -n parasol-dev | grep parasol-db

# Verify secret exists
oc get secret parasol-db -n parasol-dev

# Test database connection
oc exec -it deployment/parasol-db -n parasol-dev -- \
  psql -U parasol -d parasol -c '\dt'
----

== Argo CD Issues

=== Application Not Syncing

**Symptom:** Argo CD shows `OutOfSync`.

**Solutions:**

[source,bash]
----
# Manual sync
oc patch application parasol-dev -n parasol-dev --type=merge \
  -p '{"operation":{"initiatedBy":{"username":"admin"},"sync":{}}}'

# Or use Argo CD UI: Click "Sync" button

# Check GitOps repository branch
cd ~/parasol-gitops
git status
git log --oneline -5
----

=== Application Health Degraded

**Symptom:** Argo CD shows `Degraded` health status.

**Check:**

[source,bash]
----
# View application details
oc get application parasol-dev -n parasol-dev -o yaml

# Check resource status
oc get deployment,service,route -n parasol-dev
----

== Webhook Issues

=== Pipeline Not Triggering

**Symptom:** Git push doesn't trigger pipeline.

**Check:**

[source,bash]
----
# Verify EventListener is running
oc get pods -n parasol-dev | grep el-gitlab-listener

# Check webhook URL is accessible
oc get route el-gitlab-listener -n parasol-dev
curl http://$(oc get route el-gitlab-listener -n parasol-dev -o jsonpath='{.spec.host}')

# View EventListener logs
oc logs deployment/el-gitlab-listener -n parasol-dev --tail=50
----

**Fix:** Verify webhook is configured in GitLab with correct URL and secret.

== Quick Health Check Commands

Run these to verify overall system health:

[source,bash]
----
# All operators
oc get csv -n openshift-operators

# All running pods in parasol-dev
oc get pods -n parasol-dev

# Pipeline runs
oc get pipelinerun -n parasol-dev --sort-by=.metadata.creationTimestamp | tail -5

# Argo CD applications
oc get application -n parasol-dev

# Application health
curl http://$(oc get route parasol-app -n parasol-dev -o jsonpath='{.spec.host}')/q/health | jq .
----

== Getting Help

If issues persist:

1. Check pod logs: `oc logs <pod-name> -n parasol-dev`
2. Check events: `oc get events -n parasol-dev --sort-by='.lastTimestamp' | tail -20`
3. Verify operators: `oc get csv -n openshift-operators`
4. Ask workshop facilitator for assistance

TIP: Most issues are resolved by verifying operators are installed, pods are running, and secrets/configmaps exist.
