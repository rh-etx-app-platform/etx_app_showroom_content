== Exercise 3: Working with Multiple Branches

=== Step 1: Create Another Feature Branch

First, switch back to main:

[source,bash,role=execute]
----
git checkout main
----

Create a second feature branch:

[source,bash,role=execute]
----
git checkout -b feature/update-version
----

=== Step 2: Make Different Changes

Edit a different file (create one if needed):

[source,bash,role=execute]
----
echo "version: 1.1.0" > VERSION
git add VERSION
git commit -m "chore: bump version to 1.1.0"
----

=== Step 3: Visualize Your Branches

[source,bash,role=execute]
----
git log --oneline --graph --all --decorate
----

You should see something like:

[source,text]
----
* a3f8d92 (HEAD -> feature/update-version) chore: bump version to 1.1.0
| * b7e4c21 (feature/update-readme) docs: add contributor information to README
|/
* c9d2a45 (main) Initial commit
----

[TIP]
====
This visualization shows:

* `HEAD` points to your current branch
* Two feature branches diverged from `main`
* Each branch has its own commit history
====
