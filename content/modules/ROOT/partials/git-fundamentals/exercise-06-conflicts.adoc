== Exercise 6: Simulating Merge Conflicts

Merge conflicts occur when two branches modify the same part of a file. Let's create and resolve one.

=== Step 1: Create Conflicting Changes

First, make a change on main:

[source,bash,role=execute]
----
git checkout main
echo "# Parasol Insurance Platform" > HEADER.md
git add HEADER.md
git commit -m "docs: add header file"
----

Create a branch from an earlier point:

[source,bash,role=execute]
----
git checkout HEAD~1
git checkout -b feature/update-header
echo "# Parasol Insurance Application" > HEADER.md
git add HEADER.md
git commit -m "docs: add different header"
----

=== Step 2: Attempt to Merge

Switch to main and try to merge:

[source,bash,role=execute]
----
git checkout main
git merge feature/update-header
----

.Conflict Output
[source,text]
----
Auto-merging HEADER.md
CONFLICT (add/add): Merge conflict in HEADER.md
Automatic merge failed; fix conflicts and then commit the result.
----

=== Step 3: View the Conflict

[source,bash,role=execute]
----
cat HEADER.md
----

You'll see conflict markers:

[source,text]
----
<<<<<<< HEAD
# Parasol Insurance Platform
=======
# Parasol Insurance Application
>>>>>>> feature/update-header
----

=== Step 4: Resolve the Conflict

Edit the file to keep both changes:

[source,bash,role=execute]
----
cat > HEADER.md << 'EOF'
# Parasol Insurance Platform
## Application Suite
EOF
----

Stage the resolved file:

[source,bash,role=execute]
----
git add HEADER.md
----

Complete the merge:

[source,bash,role=execute]
----
git commit -m "merge: resolve header conflict combining both descriptions"
----

[TIP]
====
*Conflict Resolution Best Practices:*

1. Understand both changes before resolving
2. Test the merged code
3. Remove ALL conflict markers (`<<<<<<<`, `=======`, `>>>>>>>`)
4. Commit with a clear message explaining the resolution
====
