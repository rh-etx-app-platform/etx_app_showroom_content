== Exercise 2: Making Changes

=== Step 1: Create a Feature Branch

Create a new branch for your feature work:

[source,bash,role=execute]
----
git checkout -b feature/update-readme
----

.Understanding the Command
* `checkout -b` creates a new branch and switches to it
* Branch name follows convention: `feature/<description>`

Verify you're on the new branch:

[source,bash,role=execute]
----
git branch
----

You should see:
[source,text]
----
* feature/update-readme
  main
----

The asterisk (*) indicates your current branch.

=== Step 2: Make Changes to a File

Edit the README file to add your name:

[source,bash,role=execute]
----
cat >> README.md << 'EOF'

## Contributors
- Your Name ($(date +%Y-%m-%d))
EOF
----

=== Step 3: Stage and Commit Changes

View what changed:

[source,bash,role=execute]
----
git status
git diff README.md
----

Stage the changes:

[source,bash,role=execute]
----
git add README.md
----

Commit with a descriptive message:

[source,bash,role=execute]
----
git commit -m "docs: add contributor information to README"
----

.Commit Message Best Practices
[NOTE]
====
Good commit messages:

* Start with a type: `feat:`, `fix:`, `docs:`, `refactor:`
* Use imperative mood: "add feature" not "added feature"
* Keep the subject line under 50 characters
* Add detailed description if needed after a blank line

Example:
[source,text]
----
feat: add user authentication module

- Implement JWT token generation
- Add login/logout endpoints
- Include password hashing with bcrypt
----
====

=== Step 4: View Commit History

[source,bash,role=execute]
----
git log --oneline --graph --all
----

This shows a visual representation of your commit history across all branches.
